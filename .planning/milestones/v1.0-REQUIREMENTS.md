# Requirements: AutoValeur MA

**Defined:** 2026-04-11
**Core Value:** El usuario ve exactamente cómo se calcula el precio — cada factor desglosado, sin cajas negras — construyendo confianza desde el primer uso.

## v1 Requirements

### Estructura & PWA

- [x] **PWA-01**: La app existe como archivo(s) estáticos desplegables en GitHub Pages / cualquier CDN sin configuración de servidor
- [x] **PWA-02**: La app funciona completamente offline tras la primera carga (Service Worker con cache-first)
- [x] **PWA-03**: La app es instalable en la pantalla de inicio de Android (Web App Manifest completo)
- [x] **PWA-04**: La app muestra instrucciones de instalación manual para iOS Safari
- [x] **PWA-05**: La interfaz es responsive y funciona correctamente en móviles Android (pantallas 360px+)

### Navegación SPA

- [x] **NAV-01**: La app tiene 3 páginas: Estimer / Historique / Comparer con navegación via hash routing (sin recarga)
- [x] **NAV-02**: Existe una barra de navegación inferior fija con acceso a las 3 páginas
- [x] **NAV-03**: El botón atrás del navegador funciona correctamente entre páginas

### Internacionalización

- [x] **I18N-01**: La interfaz completa está disponible en francés
- [x] **I18N-02**: La interfaz completa está disponible en árabe con layout RTL correcto
- [x] **I18N-03**: El usuario puede cambiar de idioma desde la app (botón FR/AR visible)
- [x] **I18N-04**: La preferencia de idioma se persiste en localStorage
- [x] **I18N-05**: Los precios se formatean correctamente en ambos idiomas (dígitos latinos en árabe)

### Algoritmo de Valoración

- [x] **ALG-01**: El motor calcula un precio estimado basado en: marca, modelo, año, kilómetros, estado, carburant, ciudad y transmisión
- [x] **ALG-02**: El factor de edad usa la curva de depreciación calibrada para MA (tabla de 15 puntos)
- [x] **ALG-03**: El factor de kilómetros compara km reales vs media MA (15.000 km/año)
- [x] **ALG-04**: El carburant diesel tiene prima de +6% sobre essence
- [x] **ALG-05**: Las correcciones por ciudad están implementadas (Casablanca +8%, Oujda -7%, y 8 ciudades más)
- [x] **ALG-06**: El precio tiene un floor mínimo de 8.000 DH y un factor de edad mínimo de 30%
- [x] **ALG-07**: El resultado incluye rango de precio (±10%)

### Base de Datos de Coches

- [x] **DB-01**: La BD incluye al menos 14 marcas relevantes para MA: Dacia, Renault, Peugeot, Citroën, Hyundai, Kia, Toyota, VW, Ford, Fiat, Seat, Suzuki, Mercedes, BMW
- [x] **DB-02**: Cada marca incluye sus modelos más vendidos en MA (mínimo 3 modelos por marca)
- [x] **DB-03**: Cada modelo tiene años disponibles y precio base calibrado para MA
- [x] **DB-04**: Los datos están embebidos como JSON en el propio HTML (sin petición de red)

### Página Estimer

- [x] **EST-01**: El formulario permite seleccionar marca → modelo → año en cascada (el modelo filtra por marca seleccionada)
- [x] **EST-02**: El formulario incluye: kilómetros (input numérico), estado (5 opciones), carburant (5 opciones), transmisión (2 opciones), ciudad (10 opciones + otras)
- [x] **EST-03**: El resultado muestra el precio estimado de forma prominente con el rango (min-max)
- [x] **EST-04**: El resultado muestra el desglose completo del cálculo: precio base, ajuste por edad, ajuste por km, ajuste por estado, bonus carburant, corrección ciudad, factor transmisión
- [x] **EST-05**: Los ajustes del desglose muestran tanto el porcentaje como el importe en DH
- [x] **EST-06**: La valoración se guarda automáticamente en el historial local al completar la estimación
- [x] **EST-07**: Tras el resultado aparecen 3 links (Avito.ma, Moteur.ma, Wandaloo.ma) con URLs de búsqueda dinámicas para el coche valorado

### Página Historique

- [x] **HIST-01**: La página muestra todas las valoraciones guardadas, ordenadas por fecha (más reciente primero)
- [x] **HIST-02**: Cada entrada del historial muestra: marca/modelo/año, km, precio estimado y fecha
- [x] **HIST-03**: El usuario puede eliminar entradas individuales del historial
- [x] **HIST-04**: El usuario puede limpiar todo el historial con confirmación
- [x] **HIST-05**: Si el historial está vacío, se muestra un estado vacío con CTA a Estimer

### Página Comparer

- [x] **COMP-01**: El usuario puede seleccionar 2 valoraciones del historial para comparar
- [x] **COMP-02**: Las dos valoraciones se muestran lado a lado con sus datos principales
- [x] **COMP-03**: La comparación incluye barras visuales para precio, km y año
- [x] **COMP-04**: El coche con mejor ratio precio/km queda destacado visualmente como ganador (borde verde, badge)
- [x] **COMP-05**: Si hay menos de 2 valoraciones en el historial, se muestra mensaje orientativo

### Diseño Visual

- [x] **UI-01**: El diseño es moderno y limpio: paleta neutra, tipografía clara, estilo fintech/app marroquí
- [x] **UI-02**: Los colores, espaciados y radios usan CSS custom properties (--primary, --surface, --accent...)
- [x] **UI-03**: El diseño usa CSS logical properties para adaptación RTL automática
- [x] **UI-04**: Las transiciones entre páginas son suaves (no flash)

## v2 Requirements

### Mejoras futuras

- **SHARE-01**: Compartir valoración como imagen/PDF por WhatsApp
- **COMP2-01**: Exportar comparación como imagen
- **ALERT-01**: Alerta cuando un anuncio de Avito está por debajo del precio estimado
- **HIST2-01**: Editar una valoración del historial (recalcular con datos modificados)
- **DB2-01**: Actualización periódica de precios base via JSON externo opcional

## Out of Scope

| Feature | Reason |
|---------|--------|
| Backend / API | Rompe el modelo zero-cost, zero-server |
| Autenticación / cuentas | El historial es del usuario, no de la app |
| Scraping real de Avito/Moteur | Requiere servidor, viola ToS de los sitios |
| App nativa iOS/Android | La PWA cubre el caso de uso sin tiendas de apps |
| Vehículos comerciales / motos | MVP centrado en turismos; ampliar en v2 |
| Mapas / geolocalización | Innecesario — el usuario selecciona su ciudad |
| Historial en la nube / sync | Sin servidor; el historial permanece local |
