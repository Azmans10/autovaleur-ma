# Plan: 01-04 — Registrar SW y añadir meta tags PWA

**Phase:** 1 — Estructura base, SPA y PWA
**Plan:** 01-04
**Goal:** Completar el `<head>` de `index.html` con todos los meta tags PWA requeridos (iOS, Android, theme-color), registrar el Service Worker desde JS, e implementar la lógica de detección de plataforma para mostrar el banner de instalación en iOS (instrucciones manuales) y el prompt de Android (botón "Installer").

## Context

Este plan conecta los artefactos creados en 01-03 (`sw.js`, `manifest.json`, iconos) con `index.html`. Tiene dos partes:

1. **Meta tags en `<head>`**: `theme-color`, `apple-mobile-web-app-*` para iOS standalone, y `viewport` con `viewport-fit=cover` (crítico para `safe-area-inset-bottom`). El `<link rel="manifest">` ya está del plan 01-01; este plan añade las etiquetas iOS que el manifest no cubre.

2. **Registro del SW + lógica de instalación**: El SW se registra dentro de `window.addEventListener('load', ...)` para no competir con la carga inicial de la página. La lógica de instalación está bifurcada: Android usa `beforeinstallprompt`; iOS detecta el user-agent y muestra un banner con instrucciones de instalación manual (Share → Add to Home Screen).

**Prerequisitos:** Los planes 01-01, 01-02 y 01-03 deben estar completos (`index.html` con router, `sw.js`, `manifest.json`, iconos PNG).

**Requisitos implementados:** PWA-02, PWA-03, PWA-04.

**2b. Dentro del bloque `<script>` existente** (donde ya está el código del router del plan 01-02), añadir el siguiente código AL FINAL, después de `document.addEventListener('DOMContentLoaded', ...)`:

```javascript
    // ═══════════════════════════════════════════════════════════
    // SERVICE WORKER — Registro
    // Se registra en 'load' (no 'DOMContentLoaded') para no
    // competir con la carga inicial de la página.
    // ═══════════════════════════════════════════════════════════
    if ('serviceWorker' in navigator) {
      window.addEventListener('load', function() {
        navigator.serviceWorker.register('./sw.js')
          .then(function(registration) {
            console.log('AutoValeur — SW registrado, scope:', registration.scope);
          })
          .catch(function(error) {
            // El SW falla silenciosamente en file:// — normal en desarrollo sin servidor
            console.warn('AutoValeur — SW no se pudo registrar:', error.message);
          });
      });
    }

    // ═══════════════════════════════════════════════════════════
    // PWA INSTALL — Lógica bifurcada iOS / Android
    // ═══════════════════════════════════════════════════════════

    // Android Chrome: capturar el evento beforeinstallprompt
    // para mostrarlo cuando el usuario quiera instalar
    var deferredInstallPrompt = null;

    window.addEventListener('beforeinstallprompt', function(event) {
      // Prevenir que Chrome muestre el mini-infobar automático
      event.preventDefault();
      deferredInstallPrompt = event;
      console.log('AutoValeur — Install prompt capturado (Android)');
      // En fases futuras: mostrar un botón "Installer l'app" en la UI
    });

    window.addEventListener('appinstalled', function() {
      console.log('AutoValeur — App instalada correctamente');
      deferredInstallPrompt = null;
    });

    // iOS Safari: no hay beforeinstallprompt — mostrar banner con instrucciones manuales
    (function checkIOSInstall() {
      var isIOS = /iphone|ipad|ipod/i.test(navigator.userAgent);
      // window.navigator.standalone = true cuando ya está instalada como PWA en iOS
      var isInStandaloneMode = window.navigator.standalone === true
        || window.matchMedia('(display-mode: standalone)').matches;

      if (isIOS && !isInStandaloneMode) {
        var banner = document.getElementById('ios-install-banner');
        if (banner) {
          // Mostrar con pequeño delay para no bloquear la carga visual
          setTimeout(function() {
            banner.classList.add('visible');
          }, 2000);
        }
      }
    })();
```

## Notes

- **`file://` y el SW:** El Service Worker NO funciona cuando se abre el HTML directamente desde el explorador de archivos (protocolo `file://`). Siempre usar `npx http-server . -p 8080`. Si el SW no aparece en DevTools, la primera causa a verificar es que la URL sea `http://localhost:...` no `file:///...`.
- **`apple-mobile-web-app-status-bar-style: "default"`** — esta es la opción correcta para apps con header colorido. `"black-translucent"` ocultaría la status bar y requeriría compensar con padding-top adicional.
- **`beforeinstallprompt` timing** — Chrome lo dispara antes de que el SW esté activo en la primera visita. En las siguientes visitas, si la app cumple los criterios (SW activo + manifest completo), el evento se dispara en el `load`.
- **El banner iOS espera 2 segundos** para no competir visualmente con la carga de la página. Es un tradeoff entre UX y visibilidad — se puede ajustar en la Fase 6 si el testing de usuarios indica que no se ve.
- **`window.navigator.standalone`** es una propiedad propietaria de iOS Safari (no está en el estándar W3C). La combinación con `matchMedia('(display-mode: standalone)')` cubre tanto iOS como Android/Chrome.
