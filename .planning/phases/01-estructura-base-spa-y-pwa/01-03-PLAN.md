# Plan: 01-03 — Crear sw.js y manifest.json

**Phase:** 1 — Estructura base, SPA y PWA
**Plan:** 01-03
**Goal:** Crear `sw.js` (Service Worker cache-first con versioning y skip waiting) y `manifest.json` (Web App Manifest completo para instalabilidad en Android Chrome). También crear los archivos de iconos PNG placeholder necesarios para que el manifest sea válido.

## Context

Para que AutoValeur MA sea una PWA instalable y funcione offline, se necesitan exactamente tres artefactos además del `index.html`: el Service Worker, el manifest, y los iconos PNG. Este plan los crea todos.

El SW usa la estrategia cache-first: en `install` precachea todos los archivos de la app; en `fetch` sirve siempre desde caché primero (red solo como fallback para recursos no cacheados). `skipWaiting()` + `clients.claim()` garantizan que las actualizaciones se aplican en la próxima carga sin que el usuario tenga que cerrar pestañas.

Los iconos PNG son placeholders generados con Canvas — cuadrado azul con las letras "AV" en blanco. Son técnicamente válidos para que Chrome muestre el install prompt; el arte final llega en la Fase 6.

**CRÍTICO:** Los iconos del manifest deben ser archivos externos reales (PNG). No se pueden usar data URIs dentro del `manifest.json` — causa bugs en Samsung Internet y algunos Android WebView.

**Requisitos implementados:** PWA-01, PWA-02, PWA-03.

## Tasks

### Task 1: Crear sw.js (Service Worker cache-first)

**File:** `sw.js`
**Action:** create

```javascript
/*
 * AutoValeur MA — Service Worker
 * Estrategia: Cache-First con versioning
 *
 * Para actualizar la app en producción: incrementar CACHE_NAME
 * (ej: 'autovaleur-v2'). El browser detectará el cambio en sw.js
 * y reinstalará el SW, precacheando los assets actualizados.
 */

'use strict';

const CACHE_NAME = 'autovaleur-v1';

// Lista de recursos a precachear en el install.
// Incluir TODOS los archivos de la app para garantizar
// funcionamiento offline completo desde la primera carga.
const PRECACHE_URLS = [
  './',
  './index.html',
  './manifest.json',
  './icon-192.png',
  './icon-512.png',
  './apple-touch-icon.png'
];

// ── INSTALL ────────────────────────────────────────────────────
// Precachear todos los assets. skipWaiting() activa el nuevo SW
// inmediatamente sin esperar que el usuario cierre otras pestañas.
self.addEventListener('install', function(event) {
  self.skipWaiting();
  event.waitUntil(
    caches.open(CACHE_NAME).then(function(cache) {
      return cache.addAll(PRECACHE_URLS);
    })
  );
});

// ── ACTIVATE ───────────────────────────────────────────────────
// Limpiar cachés de versiones anteriores (cualquier caché cuyo
// nombre no sea CACHE_NAME se elimina). clients.claim() toma
// control de las páginas abiertas sin esperar navegación nueva.
self.addEventListener('activate', function(event) {
  event.waitUntil(
    caches.keys().then(function(cacheNames) {
      return Promise.all(
        cacheNames
          .filter(function(name) { return name !== CACHE_NAME; })
          .map(function(name) { return caches.delete(name); })
      );
    }).then(function() {
      return clients.claim();
    })
  );
});

// ── FETCH ──────────────────────────────────────────────────────
// Cache-First: responder desde caché si está disponible.
// Fallback a red para recursos no cacheados (ej: peticiones
// futuras a APIs externas). Si la red también falla, servir
// index.html como último recurso (app shell offline).
self.addEventListener('fetch', function(event) {
  // Solo interceptar peticiones GET del mismo origen
  if (event.request.method !== 'GET') return;

  event.respondWith(
    caches.match(event.request).then(function(cachedResponse) {
      if (cachedResponse) {
        return cachedResponse;
      }

      // No está en caché: ir a la red
      return fetch(event.request).then(function(networkResponse) {
        // Cachear dinámicamente si la respuesta es válida
        if (networkResponse && networkResponse.ok) {
          var responseClone = networkResponse.clone();
          caches.open(CACHE_NAME).then(function(cache) {
            cache.put(event.request, responseClone);
          });
        }
        return networkResponse;
      }).catch(function() {
        // Red no disponible: servir index.html como fallback
        return caches.match('./index.html');
      });
    })
  );
});
```

### Task 3: Generar iconos PNG placeholder con Canvas

**File:** `icon-192.png`, `icon-512.png`, `apple-touch-icon.png`
**Action:** create

Los iconos se generan ejecutando el siguiente script HTML en el browser. Crear un archivo temporal `generate-icons.html` en la raíz del proyecto, abrirlo en el browser (con el servidor http-server activo), y hacer clic en "Descargar iconos". Mover los 3 archivos descargados a la raíz del proyecto. Después eliminar `generate-icons.html`.

**Contenido de `generate-icons.html`:**

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Generador de iconos AutoValeur</title>
  <style>
    body { font-family: system-ui; padding: 2rem; background: #f0f0f0; }
    button { padding: 1rem 2rem; font-size: 1rem; background: #1a73e8; color: white; border: none; border-radius: 8px; cursor: pointer; margin-top: 1rem; display: block; }
    canvas { display: block; margin: 1rem 0; border: 1px solid #ccc; }
    p { color: #333; }
  </style>
</head>
<body>
  <h1>Generador de iconos — AutoValeur MA</h1>
  <p>Haz clic en el botón para generar y descargar los 3 iconos PNG.</p>
  <canvas id="c192" width="192" height="192"></canvas>
  <canvas id="c512" width="512" height="512"></canvas>
  <canvas id="c180" width="180" height="180"></canvas>
  <button onclick="generateAndDownload()">Descargar los 3 iconos</button>

  <script>
    function drawIcon(canvas, size) {
      var ctx = canvas.getContext('2d');
      var r = size * 0.18; // radio de las esquinas redondeadas

      // Fondo azul con esquinas redondeadas
      ctx.beginPath();
      ctx.moveTo(r, 0);
      ctx.lineTo(size - r, 0);
      ctx.quadraticCurveTo(size, 0, size, r);
      ctx.lineTo(size, size - r);
      ctx.quadraticCurveTo(size, size, size - r, size);
      ctx.lineTo(r, size);
      ctx.quadraticCurveTo(0, size, 0, size - r);
      ctx.lineTo(0, r);
      ctx.quadraticCurveTo(0, 0, r, 0);
      ctx.closePath();
      ctx.fillStyle = '#1a73e8';
      ctx.fill();

      // Letras "AV" en blanco
      ctx.fillStyle = '#ffffff';
      ctx.textAlign = 'center';
      ctx.textBaseline = 'middle';
      ctx.font = 'bold ' + Math.round(size * 0.38) + 'px system-ui, sans-serif';
      ctx.fillText('AV', size / 2, size / 2);
    }

    function downloadCanvas(canvas, filename) {
      var link = document.createElement('a');
      link.download = filename;
      link.href = canvas.toDataURL('image/png');
      document.body.appendChild(link);
      link.click();
      document.body.removeChild(link);
    }

    function generateAndDownload() {
      var c192 = document.getElementById('c192');
      var c512 = document.getElementById('c512');
      var c180 = document.getElementById('c180');

      drawIcon(c192, 192);
      drawIcon(c512, 512);
      drawIcon(c180, 180);

      // Descargar con pequeño delay entre cada uno
      downloadCanvas(c192, 'icon-192.png');
      setTimeout(function() { downloadCanvas(c512, 'icon-512.png'); }, 200);
      setTimeout(function() { downloadCanvas(c180, 'apple-touch-icon.png'); }, 400);
    }

    // Pre-renderizar para previsualizar
    window.onload = function() {
      drawIcon(document.getElementById('c192'), 192);
      drawIcon(document.getElementById('c512'), 512);
      drawIcon(document.getElementById('c180'), 180);
    };
  </script>
</body>
</html>
```

**Pasos:**
1. Crear el archivo `generate-icons.html` en la raíz del proyecto
2. Con `npx http-server . -p 8080` activo, abrir `http://localhost:8080/generate-icons.html`
3. Hacer clic en "Descargar los 3 iconos" — el browser descargará `icon-192.png`, `icon-512.png`, `apple-touch-icon.png`
4. Mover los 3 archivos descargados a la raíz del proyecto (junto a `index.html`)
5. Eliminar `generate-icons.html` (ya no se necesita)

## Notes

- **`display: "browser"` rompe la instalabilidad** — Chrome solo muestra el install prompt para `"standalone"`, `"fullscreen"` o `"minimal-ui"`. El `manifest.json` de este plan usa `"standalone"`.
- **`purpose: "any maskable"` en el icono de 192px** — permite que Android adapte la forma del icono (circular, cuadrado redondeado, etc.) según el launcher. Sin `maskable`, el icono puede aparecer con padding o recortado.
- **`sw.js` NO debe incluirse en su propio `PRECACHE_URLS`** — el browser gestiona las actualizaciones del archivo `sw.js` directamente (lo recarga cuando cambia); incluirlo en la caché interfiere con ese mecanismo.
- **Actualizar la app en el futuro:** cada vez que se modifique `index.html`, cambiar `CACHE_NAME` en `sw.js` (ej: `'autovaleur-v2'`). Sin este cambio, los usuarios seguirán viendo la versión cacheada.
- **`scope: "./"` en manifest** — garantiza que la PWA solo controla el directorio raíz, no directorios padre (relevante para GitHub Pages donde la app puede estar en un subdirectorio).
