# Plan: 01-01 — Crear index.html con estructura HTML semántica

**Phase:** 1 — Estructura base, SPA y PWA
**Plan:** 01-01
**Goal:** Crear el archivo `index.html` raíz con estructura HTML5 semántica: header, tres secciones `.page` con contenido placeholder, y barra de navegación inferior con botones de las 3 páginas. Sin CSS ni JS aún — solo la estructura.

## Context

Este es el punto de partida de toda la app. `index.html` será el único archivo HTML de AutoValeur MA — todo el CSS, JS y datos vivirán dentro de él. En esta fase, se crea el esqueleto semántico: las tres "páginas" como secciones con `hidden` nativo de HTML, la nav inferior, y los meta tags base. El archivo queda listo para que los planes 01-02 (router JS), 01-04 (SW + meta tags PWA) y 01-05 (CSS) añadan su contenido dentro del mismo archivo.

El atributo `hidden` nativo de HTML es el mecanismo de visibilidad — más semántico que `display:none` via CSS y compatible con el router hashchange del plan 01-02.

## must_haves

- [ ] El archivo `index.html` existe en la raíz del proyecto
- [ ] Contiene exactamente 3 secciones `<section class="page">` con IDs `page-estimer`, `page-historique`, `page-comparer`; solo `page-estimer` visible al cargar (las otras dos tienen el atributo `hidden`)
- [ ] La `<nav class="bottom-nav">` contiene 3 botones con `data-page="estimer"`, `data-page="historique"`, `data-page="comparer"` y texto visible
- [ ] El `<head>` incluye `charset`, `viewport` con `viewport-fit=cover`, `<title>AutoValeur MA</title>`, y `<link rel="manifest" href="manifest.json">`
- [ ] Abrir el archivo con `npx http-server . -p 8080` y visitar `http://localhost:8080` muestra la página "Estimer" con los 3 botones de navegación visibles

## Notes

- Los estilos CSS en este plan son **placeholders temporales** — el CSS definitivo se implementa en el plan 01-05. El executor del plan 01-05 debe reemplazar el bloque `<style>` completo.
- El bloque `<script>` también es placeholder — el executor de 01-02 añade el router, y el de 01-04 añade el registro del SW.
- Los SVG de la nav son inline para evitar peticiones de red externas (consistente con la arquitectura offline-first).
- Los iconos SVG usados son de la especificación Lucide (license MIT) — no requieren atribución pero es buena práctica mencionarlo.
- `viewport-fit=cover` en el meta viewport es OBLIGATORIO — sin él, `env(safe-area-inset-bottom)` devuelve siempre 0 en iPhone, lo que rompe la nav inferior en dispositivos con Dynamic Island o Home Indicator.
