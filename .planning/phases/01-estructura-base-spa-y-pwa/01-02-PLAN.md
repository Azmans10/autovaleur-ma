# Plan: 01-02 — Implementar hash router en JS

**Phase:** 1 — Estructura base, SPA y PWA
**Plan:** 01-02
**Goal:** Implementar el sistema de navegación SPA basado en hash routing dentro del bloque `<script>` de `index.html`. Al terminar, los 3 botones de la nav navegan entre páginas sin recargar la página, el botón Atrás del navegador funciona correctamente, y la página activa se marca visualmente en la nav.

## Context

La navegación SPA de AutoValeur MA se basa en `location.hash` + el evento `hashchange`. Cada página es una `<section class="page">` con un ID; el router escucha cambios de hash y muestra/oculta secciones usando el atributo nativo `hidden`. Este mecanismo es suficiente, no requiere librerías, y el botón Atrás funciona de forma nativa porque cada `location.hash =` añade una entrada al historial del browser.

Este plan modifica el bloque `<script>` que el plan 01-01 dejó como placeholder. Depende de que `index.html` exista con la estructura del plan 01-01 (3 secciones `.page` con IDs `page-estimer`, `page-historique`, `page-comparer`, y botones `.nav-item` con `data-page`).

**Requisitos implementados:** NAV-01, NAV-02 (parcial — la nav existe del plan 01-01, aquí se activa), NAV-03.

## Tasks

### Task 1: Reemplazar el bloque `<script>` placeholder con el router completo

**File:** `index.html`
**Action:** modify

Localizar el bloque `<script>` al final del `<body>` (actualmente contiene solo el `console.log` placeholder del plan 01-01) y reemplazarlo completamente con el siguiente código:

```html
  <script>
    'use strict';

    // ═══════════════════════════════════════════════════════════
    // ROUTER — Hash-based SPA navigation
    // Cada página es una <section class="page" id="page-X">.
    // La navegación usa location.hash; el botón Atrás funciona
    // de forma nativa porque cada hash push añade entrada al
    // historial del browser.
    // ═══════════════════════════════════════════════════════════

    const ROUTES = {
      '':           'estimer',
      'estimer':    'estimer',
      'historique': 'historique',
      'comparer':   'comparer'
    };

    const Router = {
      /**
       * Navegar a una página por nombre.
       * Usa location.hash = (no location.replace) para que el
       * botón Atrás funcione correctamente.
       * @param {string} page - 'estimer' | 'historique' | 'comparer'
       */
      navigate(page) {
        location.hash = page;
      },

      /**
       * Leer el hash actual y mostrar la página correspondiente.
       * Llamado en 'hashchange' y en 'DOMContentLoaded'.
       */
      _show() {
        const hash = location.hash.replace('#', '') || 'estimer';
        const pageId = ROUTES[hash] || 'estimer';

        // Ocultar todas las páginas
        document.querySelectorAll('.page').forEach(function(p) {
          p.hidden = true;
        });

        // Mostrar la página activa
        const target = document.getElementById('page-' + pageId);
        if (target) {
          target.hidden = false;
        }

        // Actualizar estado activo en la nav inferior
        document.querySelectorAll('.nav-item').forEach(function(btn) {
          btn.classList.toggle('active', btn.dataset.page === pageId);
        });
      },

      /**
       * Inicializar el router: conectar botones de nav y
       * mostrar la página correcta según el hash inicial.
       */
      init() {
        // Escuchar cambios de hash (botón Atrás incluido)
        window.addEventListener('hashchange', function() {
          Router._show();
        });

        // Conectar botones de la nav inferior
        document.querySelectorAll('.nav-item').forEach(function(btn) {
          btn.addEventListener('click', function() {
            Router.navigate(btn.dataset.page);
          });
        });

        // Mostrar la página inicial según el hash (o Estimer por defecto)
        Router._show();
      }
    };

    // ═══════════════════════════════════════════════════════════
    // INIT — Arrancar cuando el DOM esté listo
    // ═══════════════════════════════════════════════════════════
    document.addEventListener('DOMContentLoaded', function() {
      Router.init();
      console.log('AutoValeur MA — Router inicializado');
    });

  </script>
```

**Nota importante sobre el `onclick` inline del plan 01-01:** Si el plan 01-01 dejó atributos `onclick="Router.navigate('...')"` en los botones `.nav-item`, eliminarlos — el router los conecta con `addEventListener` en `Router.init()`. Tener ambos causaría doble navegación.

## Notes

- Usar `location.hash = page` (NO `location.replace('#' + page)`) — `replace` no añade entrada al historial y el botón Atrás no funcionaría (Pitfall 6 de PITFALLS.md).
- La variable `ROUTES` es un objeto simple que mapea hash → ID de sección. Los hashes no reconocidos caen a `'estimer'` como fallback seguro.
- Los `onclick` inline en el HTML son un anti-pattern cuando hay un router JS — usar siempre `addEventListener`. Si el plan 01-01 los incluyó, deben eliminarse en este paso.
- El router se expone globalmente como `window.Router` implícitamente (variable `const Router` en el scope del script). Las fases posteriores pueden llamar `Router.navigate('estimer')` desde cualquier parte del script.
- `DOMContentLoaded` es el evento correcto para inicializar el router — garantiza que los `.nav-item` y `.page` ya existen en el DOM antes de consultarlos.
