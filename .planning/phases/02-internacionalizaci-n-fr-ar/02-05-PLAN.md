**Cambio 2 — JS: funcion formatPrice (en el bloque script):**
Insertar DESPUES del modulo I18n (insertado por Plan 02) y ANTES de ROUTES/Router, la funcion:

```javascript
// ===============================================================
// FORMAT PRICE — Formateo de precios con digitos latinos
// Usa Intl.NumberFormat para respetar separadores de miles
// del locale y forzar digitos 0-9 (no arabes) via -u-nu-latn.
// ===============================================================

function formatPrice(amount, lang) {
  var locale = (lang === 'ar') ? 'ar-MA-u-nu-latn' : 'fr-MA';
  return new Intl.NumberFormat(locale, {
    style: 'decimal',
    maximumFractionDigits: 0
  }).format(amount) + ' DH';
}
```

Notas:
- `formatPrice(65000, 'fr')` devuelve `"65 000 DH"` (espacio como separador de miles FR)
- `formatPrice(65000, 'ar')` devuelve `"65.000 DH"` (punto como separador de miles AR-MA)
- La funcion es global (no dentro de un objeto) para ser facilmente llamable desde cualquier parte del codigo en fases futuras.
- El parametro `lang` permite usarla independientemente de I18n.currentLang si es necesario, pero en la practica se llamara con `I18n.currentLang`.
- D-01: NO se descargan fuentes externas. Solo system fonts.
- D-03: NO se embebe ningun subconjunto como base64.
